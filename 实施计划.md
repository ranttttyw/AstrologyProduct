## 为什么选微信小程序

| 对比项 | 订阅号 | 服务号 | 小程序 |
|--------|--------|--------|--------|
| 个人注册 | ✅ | ❌ 需企业资质 | ✅ |
| 对话体验 | 被动回复，5秒超时 | 客服消息接口 | 完全自由，无限制 |
| 前端自由度 | 几乎没有 | 几乎没有 | 完整UI，可做动画/可视化 |
| 主动推送 | 受限 | 模板消息 | 订阅消息 |
| 分享传播 | 弱 | 弱 | 原生支持分享到聊天/朋友圈 |
| 开发量 | 只需后端 | 只需后端 | 前端+后端 |
| 面试展示 | 一般 | 一般 | 很好，可直接扫码演示 |

**结论：小程序体验最好、个人可注册、面试展示效果最佳，多花几天写前端完全值得。**

```python
import httpx

async def login(code: str):
    """用小程序前端传来的code换取openid"""
    resp = await httpx.get(
        "https://api.weixin.qq.com/sns/jscode2session",
        params={
            "appid": APPID,
            "secret": APP_SECRET,
            "js_code": code,
            "grant_type": "authorization_code"
        }
    )
    data = resp.json()
    return data["openid"], data.get("session_key")
```

```python
router = APIRouter()

@router.post("/api/chat")
async def chat(request: ChatRequest):
```

### Step 3: 接入AI API（Day 2-3）

**推荐方案对比：**

| 方案 | 优点 | 缺点 | 成本 |
|------|------|------|------|
| DeepSeek (推荐) | 性价比极高、中文好、API兼容OpenAI格式 | 高峰期偶尔慢 | 极低（百万token几元） |

| Claude/GPT | 效果最好 | 需代理，合规风险 | 较高 |

**MVP建议用DeepSeek**，API格式跟OpenAI兼容，以后换模型也方便。

**AI对话模块：**
```python
# services/ai.py
import httpx

DEEPSEEK_API_KEY = "your_key"
DEEPSEEK_URL = "https://api.deepseek.com/chat/completions"

async def chat_with_ai(user_message, user_chart, today_transits, chat_history=None):
    system_prompt = build_system_prompt(user_chart, today_transits)
    
    messages = [{"role": "system", "content": system_prompt}]
    
    # 加入最近5轮对话历史（实现记忆）
    if chat_history:
        for h in chat_history[-10:]:  # 最近5轮=10条消息
            messages.append(h)
    
    messages.append({"role": "user", "content": user_message})
    
    async with httpx.AsyncClient(timeout=30) as client:
        resp = await client.post(
            DEEPSEEK_URL,
            headers={"Authorization": f"Bearer {DEEPSEEK_API_KEY}"},
            json={
                "model": "deepseek-chat",
                "messages": messages,
                "max_tokens": 800,
                "temperature": 0.8  # 稍高一点，让回复更有个性
            }
        )
    return resp.json()["choices"][0]["message"]["content"]
```

**产出：** AI对话引擎可用，传入用户信息就能生成个性化回复

**核心代码：**
```python
# services/astrology.py
from flatlib.chart import Chart
from flatlib.datetime import Datetime
from flatlib.geopos import GeoPos
from flatlib import const

# 主要城市经纬度（先覆盖一线+主要城市，后续可接地图API）
CITY_COORDS = {
    "北京": (39.9042, 116.4074),
    "上海": (31.2304, 121.4737),
    "广州": (23.1291, 113.2644),
    "深圳": (22.5431, 114.0579),
    "杭州": (30.2741, 120.1551),
    "成都": (30.5728, 104.0668),
    "武汉": (30.5928, 114.3055),
    "南京": (32.0603, 118.7969),
    # ... 可以继续添加更多城市
}

def calculate_chart(birth_date, birth_time, city):
    """
    计算用户星盘
    birth_date: "1998/03/15"
    birth_time: "10:30"
    city: "北京"
    """
    dt = Datetime(birth_date, birth_time, '+8:00')  # 中国时区UTC+8
    lat, lon = CITY_COORDS.get(city, (39.9042, 116.4074))  # 默认北京
    pos = GeoPos(lat, lon)
    chart = Chart(dt, pos)
    
    return {
        "sun": {"sign": str(chart.get(const.SUN).sign), "degree": round(chart.get(const.SUN).lon % 30, 1)},
        "moon": {"sign": str(chart.get(const.MOON).sign), "degree": round(chart.get(const.MOON).lon % 30, 1)},
        "ascendant": {"sign": str(chart.get(const.ASC).sign)},
        "mercury": {"sign": str(chart.get(const.MERCURY).sign)},
        "venus": {"sign": str(chart.get(const.VENUS).sign)},
        "mars": {"sign": str(chart.get(const.MARS).sign)},
        "jupiter": {"sign": str(chart.get(const.JUPITER).sign)},
        "saturn": {"sign": str(chart.get(const.SATURN).sign)},
    }

def get_today_transits():
    """获取今日行星位置和重要相位"""
    from datetime import datetime
    now = datetime.now()
    dt = Datetime(now.strftime('%Y/%m/%d'), now.strftime('%H:%M'), '+8:00')
    pos = GeoPos(39.9042, 116.4074)  # 行运盘位置不影响行星位置
    chart = Chart(dt, pos)
    
    transits = {}
    for planet in [const.SUN, const.MOON, const.MERCURY, const.VENUS, 
                   const.MARS, const.JUPITER, const.SATURN]:
        p = chart.get(planet)
        transits[planet] = {
            "sign": str(p.sign),
            "degree": round(p.lon % 30, 1),
            "retrograde": p.movement() == "Retrograde"  # 是否逆行
        }
    
    return transits

def format_chart_for_prompt(chart_data):
    """把星盘数据格式化成AI能理解的文字"""
    sign_cn = {
        "Aries": "白羊座", "Taurus": "金牛座", "Gemini": "双子座",
        "Cancer": "巨蟹座", "Leo": "狮子座", "Virgo": "处女座",
        "Libra": "天秤座", "Scorpio": "天蝎座", "Sagittarius": "射手座",
        "Capricorn": "摩羯座", "Aquarius": "水瓶座", "Pisces": "双鱼座"
    }
    
    lines = [
        f"太阳星座：{sign_cn.get(chart_data['sun']['sign'], chart_data['sun']['sign'])}",
        f"月亮星座：{sign_cn.get(chart_data['moon']['sign'], chart_data['moon']['sign'])}",
        f"上升星座：{sign_cn.get(chart_data['ascendant']['sign'], chart_data['ascendant']['sign'])}",
        f"水星：{sign_cn.get(chart_data['mercury']['sign'], chart_data['mercury']['sign'])}",
        f"金星：{sign_cn.get(chart_data['venus']['sign'], chart_data['venus']['sign'])}",
        f"火星：{sign_cn.get(chart_data['mars']['sign'], chart_data['mars']['sign'])}",
    ]
    return "\n".join(lines)
```

**产出：** 输入生日信息→输出完整星盘数据 + 今日行运

### Step 5: 核心Prompt工程（Day 4-5）⭐ 最重要的一步

**这是产品成败的关键。** AI回复的质量直接决定用户留不留下来。

**System Prompt：**

```
你是"星语"，一个温暖、有洞察力的星座顾问。你像一个懂占星的好朋友，
用亲切自然的语气和用户聊天。

## 你的身份
- 你精通占星学，熟悉每个星座、行星、宫位、相位的含义
- 你说话温暖但不油腻，有洞察力但不故弄玄虚
- 你把星座知识转化为用户能理解和行动的建议
- 你始终强调"星象是参考，决定权在你自己"

## 回答规则
1. 回答控制在150-300字，不要太长
2. 先共情用户的困扰（1-2句），让用户感到被理解
3. 再结合星盘和今日星象分析（用通俗语言，不堆术语）
4. 最后给出1-2条具体可操作的建议
5. 用温暖鼓励的话收尾
6. emoji控制在1-3个
7. 绝不涉及健康诊断、法律建议、投资建议
8. 如果用户没有设置星盘，用太阳星座做基础分析，并温柔提醒补全信息

## 用户星盘信息
{user_chart}

## 今日星象
{today_transits}

## 语气示范

好的回答示例：
用户问："我今天要不要跟面试官发follow up邮件？"
"理解你的纠结，发还是不发确实让人犹豫～

你的水星在天秤座，天生就很擅长拿捏沟通的分寸感。
今天水星和木星形成六合相位，是特别利好表达和争取机会的一天。

建议你大胆发，但用你天秤水星的风格——
礼貌、得体、不卑不亢，表达对这个机会的兴趣就好。
今天的星象会帮你的 ✨"

不好的回答示例：
"根据星象分析，今日水星与木星呈60度六合相位，
利好第三宫沟通事务，建议在沟通方面积极主动。"
（太冷、太术语化、没有共情、没有具体建议）
```

**Prompt调优方法：**
1. 自己先测试20-30个不同类型的问题（感情、工作、学业、日常选择）
2. 找5-10个朋友试用，让他们打分（1-5分：共鸣感、实用性、趣味性）
3. 把不好的回复收集起来，分析问题出在哪，针对性调整Prompt

**产出：** 一套经过调优的Prompt，AI回复质量达到可用水平

### Step 6: 数据库设计（Day 5）

**选型：** SQLite（MVP够用，零配置，一个文件搞定）

```python
# models/database.py
import sqlite3
import json
from datetime import datetime

DB_PATH = "astrology.db"

def init_db():
    conn = sqlite3.connect(DB_PATH)
    conn.executescript("""
        CREATE TABLE IF NOT EXISTS users (
            openid TEXT PRIMARY KEY,
            nickname TEXT,
            birth_date TEXT,
            birth_time TEXT,
            birth_city TEXT,
            chart_data TEXT,          -- JSON格式的星盘数据
            created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
            last_active TIMESTAMP
        );
        
        CREATE TABLE IF NOT EXISTS messages (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            openid TEXT,
            role TEXT,                -- 'user' 或 'assistant'
            content TEXT,
            message_type TEXT,        -- 'decision'决策/'horoscope'运势/'chat'闲聊
            created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
            FOREIGN KEY (openid) REFERENCES users(openid)
        );
        
        CREATE TABLE IF NOT EXISTS daily_horoscope_cache (
            date TEXT,
            openid TEXT,
            content TEXT,
            created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
            PRIMARY KEY (date, openid)
        );
    """)
    conn.close()
```

**产出：** 数据库结构就绪，支持用户档案 + 聊天记录 + 运势缓存

```javascript
// pages/chat/chat.js
const api = require('../../utils/api')

Page({
  data: {
    messages: [],
    inputText: '',
    isLoading: false,
    hasChart: false,
    scrollToId: ''
  },

  onLoad() {
    this.checkUserChart()
  },

  async checkUserChart() {
    const profile = await api.getUserProfile()
    this.setData({ hasChart: !!profile.chart_data })
  },

  onInput(e) {
    this.setData({ inputText: e.detail.value })
  },

  async sendMessage() {
    const text = this.data.inputText.trim()
    if (!text || this.data.isLoading) return

    // 添加用户消息
    const userMsg = { id: Date.now(), role: 'user', content: text }
    this.setData({
      messages: [...this.data.messages, userMsg],
      inputText: '',
      isLoading: true,
      scrollToId: `msg-loading`
    })

    try {
      // 调用后端API
      const reply = await api.chat(text, this.getRecentHistory())
      
      const botMsg = { id: Date.now() + 1, role: 'bot', content: reply }
      this.setData({
        messages: [...this.data.messages, botMsg],
        isLoading: false,
        scrollToId: `msg-${botMsg.id}`
      })
    } catch (err) {
      // 错误处理
      const errorMsg = { id: Date.now() + 1, role: 'bot', content: '星象信号有点不稳定，请稍后再试～' }
      this.setData({
        messages: [...this.data.messages, errorMsg],
        isLoading: false
      })
    }
  },

  quickAsk(e) {
    this.setData({ inputText: e.currentTarget.dataset.q })
    this.sendMessage()
  },

  getRecentHistory() {
    // 返回最近5轮对话
    return this.data.messages.slice(-10).map(m => ({
      role: m.role === 'bot' ? 'assistant' : 'user',
      content: m.content
    }))
  },

  goToProfile() {
    wx.switchTab({ url: '/pages/profile/profile' })
  }
})
```

### Step 12: API请求封装 + 登录（Day 11）

```javascript
// utils/api.js
const BASE_URL = 'https://your-domain.com'

let token = ''

async function login() {
  const { code } = await wx.login()
  const res = await request('/api/login', { code })
  token = res.token
  return res
}

function request(path, data = {}, method = 'POST') {
  return new Promise((resolve, reject) => {
    wx.request({
      url: BASE_URL + path,
      method,
      data,
      header: { 'Authorization': `Bearer ${token}` },
      success: (res) => resolve(res.data),
      fail: reject
    })
  })
}

module.exports = {
  login,
  chat: (message, history) => request('/api/chat', { message, history }),
  getUserProfile: () => request('/api/user/profile', {}, 'GET'),
  bindChart: (data) => request('/api/user/bindChart', data),
  getTodayHoroscope: () => request('/api/horoscope/today', {}, 'GET'),
}
```

**技术判断：** "选小程序而非App，因为开发快、获客成本低、微信生态自带传播。选AI而非人工编辑，因为个性化能力和零边际成本。"

**Agent角度：** "这不是套壳ChatGPT。它具备AI agent的三个核心特征：持久记忆（星盘档案+对话历史）、工具调用（实时天文历计算）、个性化推理（综合多维数据生成建议）。"

**数据：** "MVP 3周上线，内测30人，7日留存X%，日均对话Y轮。发现Z是最大问题，迭代后指标提升N%。"
