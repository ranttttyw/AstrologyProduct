# 星语 · 微信星座决策助手 — 产品需求文档（PRD）

**版本**: v1.0  
**日期**: 2026年4月10日  
**作者**: 总裁

---

## 1. Problem Statement

年轻人在日常生活中频繁面临大大小小的决策——要不要投这家公司的简历、今天要不要跟crush表白、周末选哪个活动参加。很多人习惯性地刷星座运势来寻找心理支撑，但现有的星座产品（测测星座、准了等）提供的是千篇一律的通用运势（"今日感情运★★★★"），无法针对用户的具体问题给出个性化建议。

用户真正需要的不是"今天运势如何"，而是"我今天这个具体的事儿该不该做"。

这个gap意味着：用户不得不自己把通用运势"翻译"成对自己有用的行动指南，或者去付费找占星师做一对一咨询（单次50-300元）。一个能在微信聊天框里随时对话、结合用户星盘和当日星象给出具体决策建议的AI助手，可以填补这个空白。

---

## 2. Goals

**用户目标：**
- 让用户在30秒内就一个具体问题获得结合星象的个性化建议，降低决策焦虑
- 通过每日个性化运势推送，帮助用户养成"每天打开看一眼"的习惯

**产品目标：**
- MVP上线后30天内获得500+关注用户，DAU达到15%（75人）
- 用户7日留存率达到30%以上
- 形成可复述的产品故事，用于PM实习面试展示

**商业目标（长期）：**
- 验证付费意愿：上线付费功能后，付费转化率达到3-5%
- 单用户月均使用频次达到8次以上

---

## 3. Non-Goals（v1不做）

- **不做小程序或独立App**：v1仅做微信公众号对话形态，降低开发成本，快速验证核心假设。等验证通过后再考虑更丰富的前端交互。
- **不做社交/社区功能**：不做星座匹配、合盘、用户社区等功能。v1专注于单用户的决策助手体验。
- **不做专业占星教学**：不教用户看星盘、学占星。产品定位是"用得上"而非"学得会"。
- **不做塔罗/其他占卜形式**：v1聚焦星座星盘体系，保持产品调性一致。
- **不做变现**：v1阶段全部免费，先跑通体验和留存，再考虑付费模型。

## 4. Target Users

**核心用户画像：18-30岁年轻人，对星座有基础认知和兴趣**

典型特征：
- 知道自己的太阳星座，可能还知道月亮和上升，但看不太懂完整星盘
- 日常会在社交媒体上刷星座相关内容，把星座当社交话题和生活调味剂
- 面对决策时有一定的"选择困难"倾向，喜欢找外部参考来帮自己下决心
- 不把星座当"算命"信，但觉得"还挺准的"，心态偏向趣味和心理慰藉
- 高频使用微信，习惯在微信内完成大部分信息获取

**次要用户：对星座有较深了解的爱好者**
- 看得懂星盘，关注行星逆行、新月满月等星象事件
- 对解读的专业度有更高要求
- 可能成为高付费意愿用户和口碑传播者

## 5. User Stories

**新用户上手：**

- 作为一个刚关注公众号的用户，我想快速了解这个bot能做什么，这样我知道怎么开始使用它
- 作为一个新用户，我想输入我的生日和出生时间来生成星盘档案，这样bot以后的回答能结合我的个人星盘
- 作为一个不知道自己出生时间的用户，我也想能使用基本功能，这样我不会因为缺少信息而被挡在门外

**日常运势：**

- 作为一个每天早上想看运势的用户，我想收到一条个性化的今日运势消息，这样我对今天的节奏有个心理预期
- 作为一个忙碌的用户，我想运势消息简洁明了、有重点提示，这样我30秒内就能看完并记住关键信息

**决策咨询（核心场景）：**

- 作为一个正在纠结要不要做某件事的用户，我想在聊天框里描述我的问题并得到结合星象的建议，这样我能获得一个新的视角来帮助自己做决定
- 作为一个不确定今天适不适合做某事的用户，我想知道当日星象对这件事是利好还是需要谨慎，这样我能选择更合适的时机
- 作为一个希望得到情绪支持的用户，我想要bot的回答是温暖有共鸣的、而不是冷冰冰的信息罗列，这样我在犹豫焦虑时能得到安慰

**星盘解读：**

- 作为一个对自己星盘好奇的用户，我想问关于自己星盘的问题（比如"我的水星天秤代表什么"），这样我能更了解自己的性格特质

## 6. Requirements

### Must-Have（P0）— MVP必须包含

**6.1 用户星盘档案建立**
- 用户发送生日信息（日期 + 出生时间 + 出生城市）后，系统自动计算并存储星盘核心数据（太阳、月亮、上升、水星、金星、火星的星座和宫位）
- 如果用户不知道出生时间，仅根据出生日期提供太阳星座的基础服务，并提示补全信息可以获得更精准的解读
- 验收标准：用户输入"1998年3月15日 上午10:30 北京"后，系统在10秒内返回正确的星盘摘要

**6.2 每日个性化运势推送**
- 每天早上8:00根据用户的星盘数据和当日真实星象（行星位置、相位），自动推送一条个性化运势
- 运势内容包含：整体能量概述、今日关键提示（1-2句可操作建议）、适合做/不太适合做的事
- 验收标准：同一天内，白羊座太阳+天蝎月亮的用户和双鱼座太阳+狮子月亮的用户收到的运势内容明显不同

**6.3 决策咨询对话**
- 用户在聊天框中描述一个具体问题，bot结合用户星盘 + 当日星象生成一段个性化的建议
- 回答需包含：相关星象依据（用通俗语言解释）、针对这个问题的具体建议、温暖鼓励的tone
- 回答长度控制在200-400字，避免过长导致阅读负担
- 验收标准：用户问"我今天要不要跟面试官发follow up邮件"，bot能结合用户水星位置和当日水星相位给出有针对性的回答

**6.4 公众号消息架构**
- 用户关注公众号后收到欢迎消息，引导设置星盘档案
- 支持异步回复机制：先返回"正在为你推算..."的即时消息，再通过客服接口发送AI生成的完整回答
- 验收标准：从用户发送消息到收到最终回复不超过15秒

### Nice-to-Have（P1）— 快速迭代加入

**6.5 星盘基础问答**
- 用户可以问关于自己星盘的问题，如"我的月亮天蝎是什么意思""我的金星在第七宫代表什么"
- bot用通俗易懂的语言解读，避免过度专业术语

**6.6 重要星象事件提醒**
- 在水逆、新月、满月等重要星象事件前推送提醒，说明对用户个人星盘的具体影响
- 比如："本周三水星开始逆行，因为你的水星在双子座，沟通方面需要特别注意确认信息"

**6.7 历史对话记忆**
- bot能记住用户之前提过的问题和背景，在后续对话中给出更有上下文的建议
- 比如用户之前问过求职相关的问题，下次再问时bot能说"上次你提到在考虑投A公司..."

### Future Considerations（P2）— v2+考虑

**6.8 好友合盘**：用户邀请好友，生成两人的星座配对分析

**6.9 付费深度报告**：月运/年运PDF报告、职业方向深度分析等

**6.10 小程序版本**：星盘可视化、抽牌动画、历史记录查看等更丰富的前端体验

**6.11 多语言支持**：英文版本，拓展海外华人或对占星感兴趣的国际用户

## 8. 核心交互流程

### Flow 1: 新用户注册

```
用户关注公众号
    ↓
Bot发送欢迎消息："嗨～我是星语，你的专属星座决策助手✨ 
    告诉我你的生日，我来帮你看看星盘吧！
    格式：1998年3月15日 10:30 北京"
    ↓
用户发送生日信息
    ↓
Bot计算星盘，返回星盘摘要：
    "收到！你的星盘档案已建好：
     ☀️ 太阳双鱼 — 感性细腻的灵魂
     🌙 月亮狮子 — 内心其实很要强
     ⬆️ 上升天秤 — 给人温和好相处的第一印象
     明天开始你就能收到专属运势了～
     现在也可以直接问我问题哦，比如'我今天适合去面试吗？'"
```

### Flow 2: 每日运势推送

```
每天早上8:00自动推送
    ↓
"早安～今天的星象能量对你来说：

🔮 整体：月亮进入你的第十宫，事业领域能量很足
💡 关键提示：今天特别适合展示自己的能力，
   你的太阳双鱼的直觉力+月亮狮子的表现力是很好的组合
✅ 适合：主动联系、展示成果、社交拓展
⚠️ 谨慎：重大财务决策（金星和土星有紧张相位）

有什么纠结的事情？随时问我～"
```

### Flow 3: 决策咨询（核心场景）

```
用户："我在纠结要不要接这个外包项目，钱还行但会很忙"
    ↓
Bot："正在结合今日星象为你分析..."（即时回复）
    ↓
Bot（5-10秒后）：
    "关于要不要接这个外包项目，我结合你的星盘和今天的星象来看：

     你的太阳双鱼+月亮狮子意味着你很容易被'帮别人'的感觉打动，
     但月亮狮子也说明你内心其实很在意自己的时间和舞台。

     今天水星和木星形成六合相位，这是一个利好沟通和商业谈判的星象。
     如果你决定接，今天是一个适合谈条件的好时机——
     建议你主动提出你的边界（比如时间范围、修改次数），
     不要不好意思开口，木星的能量会帮你。

     如果你其实内心更想把时间留给自己的事，
     那也完全没问题，听听月亮狮子的声音。

     总结：星象支持你今天做沟通和谈判，但'接不接'这个决定，
     你心里其实已经有答案了～ 你倾向哪个？"
```

## 9. 技术方案概述

| 模块 | 方案 | 说明 |
|------|------|------|
| 前端 | 微信公众号（服务号） | 支持客服消息接口，可异步推送 |
| 后端 | Python (Flask/FastAPI) | 轻量快速，星座计算库生态好 |
| AI引擎 | Claude API / 通义千问 API | 国内部署建议用通义千问避免合规问题 |
| 星历计算 | flatlib / Swiss Ephemeris | 开源，精确计算行星位置和相位 |
| 数据库 | SQLite → PostgreSQL | MVP用SQLite，用户量上来后迁移 |
| 部署 | 云服务器（阿里云/腾讯云） | 学生优惠，成本可控 |
| 定时任务 | Celery / APScheduler | 每日运势推送和星象数据更新 |

### 核心Prompt工程

产品的核心竞争力在于AI回复的质量。需要精心设计system prompt，包含：
- 角色设定：温暖、有洞察力、像一个懂星座的好朋友
- 星座知识库：各星座特质、行星含义、相位解读规则
- 回答结构：先共情用户的困扰 → 结合星盘和星象分析 → 给出具体可操作的建议 → 温暖收尾
- 语气要求：不说教、不绝对化、始终强调"最终决定权在你"
- 合规边界：不做健康、法律、投资等专业领域的决策建议

## 10. Open Questions

| 问题 | 负责方 | 是否blocking |
|------|--------|-------------|
| 微信服务号需要企业资质注册，是否用个人订阅号先验证？订阅号不支持客服消息接口，需要其他异步方案 | 产品/技术 | Yes |
| AI API选用国内（通义千问/文心一言）还是海外（Claude/GPT）？涉及成本、效果和合规 | 技术 | Yes |
| 每日运势推送的时间是否固定8:00？是否需要让用户自定义？ | 产品 | No |
| 星盘计算的精确度要求多高？是否需要考虑时区、夏令时等边缘case？ | 技术 | No |
| 星座内容是否需要专业占星师审核？还是完全依赖AI生成？ | 产品/内容 | No |
| 是否需要设计分享机制（把运势或建议分享到朋友圈）来帮助获客？ | 产品/增长 | No |

## 11. Timeline Considerations

### Phase 1: MVP（2-3周）
- Week 1：搭建后端架构 + 星盘计算模块 + AI prompt调优
- Week 2：对接微信公众号API + 完成核心对话流程
- Week 3：内测优化 + 邀请20-30个朋友试用反馈

### Phase 2: 优化迭代（第4-6周）
- 根据用户反馈优化回复质量和交互体验
- 加入P1功能（星盘问答、星象事件提醒）
- 开始小范围推广（小红书、朋友圈）

### Phase 3: 增长验证（第7-12周）
- 数据分析，验证留存和使用频次指标
- 探索付费模型
- 准备PM面试的完整产品叙事

## 12. 竞品对比与差异化

| 维度 | 测测星座 | 准了 | 星语（本产品） |
|------|----------|------|----------------|
| 形态 | 独立App | 独立App | 微信公众号对话 |
| 运势内容 | 通用运势，按星座分类 | 通用运势 + 部分个性化 | 完全基于个人星盘的AI个性化生成 |
| 决策支持 | 无 | 无 | 核心功能：用户问具体问题获得建议 |
| 交互方式 | 浏览内容 | 浏览内容 | 自然语言对话 |
| 获客成本 | 需要下载App | 需要下载App | 扫码关注即用，零门槛 |
| 内容生产 | 编辑人工撰写 | 编辑/算法生成 | AI实时生成，边际成本趋近于零 |

**核心差异化：从"看运势"到"问决策"的场景升级，以及微信原生对话的零门槛体验。**
