# 星语心事 · AI星座情绪陪伴助手 — 产品需求文档（PRD）

**版本**: v2.0（Web版）  
**日期**: 2026年4月22日  
**作者**: 总裁

---

## 1. Problem Statement

年轻人在日常生活中频繁面临大大小小的决策——要不要投这家公司的简历、今天要不要跟crush表白、周末选哪个活动参加。很多人习惯性地刷星座运势来寻找心理支撑，但现有的星座产品（测测星座、准了等）提供的是千篇一律的通用运势（"今日感情运★★★★"），无法针对用户的具体问题给出个性化建议。

用户真正需要的不是"今天运势如何"，而是"我今天这个具体的事儿该不该做"。

这个gap意味着：用户不得不自己把通用运势"翻译"成对自己有用的行动指南，或者去付费找占星师做一对一咨询（单次50-300元）。一个能随时对话、结合用户星盘和当日星象给出具体决策建议的AI助手，可以填补这个空白。

---

## 2. Goals

**用户目标：**
- 让用户在30秒内就一个具体问题获得结合星象的个性化建议，降低决策焦虑
- 通过每日个性化运势，帮助用户养成"每天打开看一眼"的习惯

**产品目标：**
- 上线后30天内获得500+注册用户，DAU达到15%
- 用户7日留存率达到30%以上

**商业目标（长期）：**
- 验证付费意愿：上线付费功能后，付费转化率达到3-5%
- 单用户月均使用频次达到8次以上

---

## 3. Non-Goals

- **不做原生App**：当前使用Web应用形态，通过浏览器即可访问，降低获客门槛
- **不做社交/社区功能**：不做星座匹配、合盘、用户社区等功能，专注于单用户的决策助手体验
- **不做专业占星教学**：产品定位是"用得上"而非"学得会"
- **不做塔罗/其他占卜形式**：聚焦星座星盘体系，保持产品调性一致
- **不做变现**：当前阶段全部免费，先跑通体验和留存

---

## 4. Target Users

**核心用户画像：18-30岁年轻人，对星座有基础认知和兴趣**

典型特征：
- 知道自己的太阳星座，可能还知道月亮和上升，但看不太懂完整星盘
- 日常会在社交媒体上刷星座相关内容，把星座当社交话题和生活调味剂
- 面对决策时有一定的"选择困难"倾向，喜欢找外部参考来帮自己下决心
- 不把星座当"算命"信，但觉得"还挺准的"，心态偏向趣味和心理慰藉

**次要用户：对星座有较深了解的爱好者**
- 看得懂星盘，关注行星逆行、新月满月等星象事件
- 对解读的专业度有更高要求
- 可能成为高付费意愿用户和口碑传播者

---

## 5. User Stories

**新用户上手：**
- 作为一个首次访问的用户，我打开网页就能直接使用，不需要注册登录，降低门槛
- 作为一个新用户，我想输入生日、出生时间和城市来生成星盘档案，这样AI的回答能结合我的个人星盘
- 作为一个不知道自己出生时间的用户，我也想能使用基本功能

**日常运势：**
- 作为一个每天想看运势的用户，我想看到一条结合个人星盘的今日运势，包含多维度评分
- 作为一个忙碌的用户，我想运势简洁明了，30秒内就能看完

**决策咨询（核心场景）：**
- 作为一个纠结的用户，我想在聊天框里描述我的问题并得到结合星象的建议
- 作为一个希望得到情绪支持的用户，我想要AI像朋友一样有温度地聊天，而不是冷冰冰的信息罗列

**个人资料：**
- 作为一个用户，我想设置昵称、性别、职业，让AI更了解我

## 6. Requirements

### Must-Have（P0）

**6.1 访客自动登录**
- 用户打开网页自动生成访客身份，无需注册登录
- Token持久化，刷新页面不丢失身份

**6.2 用户星盘档案建立**
- 用户输入生日、出生时间、出生城市后，系统自动计算并存储星盘数据（太阳、月亮、上升、水金火木土天海冥）
- 未设置星盘时，聊天页面提示用户先设置

**6.3 每日个性化运势**
- 根据用户星盘和当日真实星象生成个性化运势
- 运势包含：综合/爱情/事业/财运/健康五维评分、幸运色、幸运数字、贴士
- 当日缓存，同一天不重复生成

**6.4 AI决策咨询对话**
- 用户发送消息，AI结合用户星盘+当日星象+用户资料+对话历史生成个性化回复
- 回复风格像朋友聊天：有主见、有情绪、长短不一
- RAG知识库增强：根据问题语义检索星座知识片段注入Prompt

**6.5 用户资料管理**
- 支持设置昵称、性别（影响AI称呼语气）、职业（影响AI建议角度）
- 星盘信息可修改

### Nice-to-Have（P1）— 后续迭代

**6.6 重要星象事件提醒**：水逆、新月、满月等事件的个性化影响解读

**6.7 分享功能**：将运势或对话截图分享到社交平台

**6.8 PWA支持**：添加到桌面，获得类App体验

## 7. 技术方案

| 模块 | 方案 | 说明 |
|------|------|------|
| 前端 | React 18 SPA（CDN加载） | 单HTML文件，无需构建工具，通过Babel standalone编译JSX |
| 后端 | Python FastAPI | 同时托管API和前端静态文件 |
| AI引擎 | DeepSeek API | 性价比高、中文好、API兼容OpenAI格式 |
| RAG | jieba + TF-IDF（ChromaDB可选） | 星座知识库语义检索，增强AI回复专业性 |
| 星历计算 | Kerykeion + Swiss Ephemeris（含纯数学fallback） | 精准计算行星位置和相位 |
| 数据库 | SQLite | 轻量零配置 |
| 部署 | Render（免费tier） | 一站式部署，支持Python |
| 认证 | JWT + 访客UUID | 无需注册，自动登录 |

### 核心AI Pipeline

```
用户消息
    ↓
RAG检索相关星座知识（jieba分词 + TF-IDF余弦相似度）
    ↓
构建System Prompt = 基础人设 + 用户星盘 + 今日星象 + 用户资料(性别/职业) + RAG知识
    ↓
拼接最近3轮对话历史
    ↓
调用DeepSeek API (temperature=0.95, frequency_penalty=1.2)
    ↓
返回AI回复 + 保存聊天记录
```

## 9. 竞品对比与差异化

| 维度 | 测测星座 | 准了 | 星语心事（本产品） |
|------|----------|------|----------------|
| 形态 | 独立App | 独立App | Web应用，打开即用 |
| 运势内容 | 通用运势，按星座分类 | 通用运势 + 部分个性化 | 完全基于个人星盘的AI个性化生成 |
| 决策支持 | 无 | 无 | 核心功能：用户问具体问题获得建议 |
| 交互方式 | 浏览内容 | 浏览内容 | 自然语言对话，像跟朋友聊天 |
| 获客成本 | 需要下载App | 需要下载App | 分享链接即用，零门槛 |
| 内容生产 | 编辑人工撰写 | 编辑/算法生成 | AI实时生成，边际成本趋近于零 |

**核心差异化：从"看运势"到"问决策"的场景升级，以及Web即开即用的零门槛体验。**
